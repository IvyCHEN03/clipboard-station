# X Launch Kit

This kit keeps the launch focused on one promise: Linggan catches scattered ideas before they disappear.

## Recommended Post Order

1. Publish the English post with the 15-second English teaser attached.
2. Put the GitHub link in the first reply so the main post stays conversational.
3. Add the longer product demo in a second reply only when someone asks for more detail.
4. Publish the Chinese version 12-24 hours later instead of posting both languages at once.
5. During the first hour, reply to real questions with specific product details instead of repeating the launch link.

## Profile And Repository

- Pin the launch post for at least one week.
- Use the repository URL as the profile website during launch week.
- Set the repository social preview to [`linggan-github-social-preview.png`](assets/social/linggan-github-social-preview.png).
- Suggested topics: `macos`, `swiftui`, `clipboard-manager`, `productivity`, `ai-tools`, `image-downloader`, `local-first`.
- Keep the README's 15-second demo above the installation instructions.

## Reply Material

Use short, concrete replies instead of feature lists:

```text
The core workflow is local. Clipboard data is encrypted on disk, and nothing is uploaded unless optional AI tagging is enabled.
```

```text
Cmd-click the bubble on a supported image-heavy post, double-click the captured row, choose the images, and export a clean PNG set.
```

```text
Repeated copies stay as separate snippets. Search and tag filters narrow what bulk actions can change.
```

## What To Measure

Check after 2 hours, 24 hours, and 7 days:

- Video views and completion rate.
- Profile visits divided by post impressions.
- GitHub visits and stars attributed to X.
- Replies that name a real workflow or missing feature.

Do not optimize only for impressions. The useful signal is whether viewers understand the workflow, visit the repository, and star or try it.
